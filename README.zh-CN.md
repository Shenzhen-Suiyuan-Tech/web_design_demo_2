# Single Egg — 前端展示页面

本项目是 **深圳随元科技有限公司** 的对外前端设计展示页。  
通过电影级的滚动动画、逐帧 Canvas 序列与编排式排版，呈现一段完整的视觉叙事体验。

> **在线预览**: [GitHub Pages](https://Shenzhen-Suiyuan-Tech.github.io/web_design_demo_2/) (部署后可用)

---

## 项目概述

本项目基于 **Vite + TypeScript + GSAP** 构建，是一个面向外部的设计展示型单页网站。整个页面由用户滚动驱动，包含以下核心体验：

- **6 个沉浸式章节**，涵盖固定（pin）与非固定滚动行为
- **Canvas 逐帧图像擦洗**（共 630+ 帧，分布在 3 个序列中）
- **GSAP ScrollTrigger** 实现精确的滚动联动动画
- **实时文字编排** —— 位置、透明度、模糊度与裁剪路径的平滑过渡
- **响应式布局**，针对移动端与窄屏视口做了专门适配
- **自适应导航栏**，根据当前章节背景自动切换深色/浅色主题

## 环境搭建

### 1. 克隆仓库

```bash
git clone https://github.com/Shenzhen-Suiyuan-Tech/web_design_demo_2.git
cd web_design_demo_2
```

### 2. 安装依赖

```bash
npm install
```

### 3. 添加帧序列

项目需要三组预渲染的图像序列用于 Canvas 动画。由于文件体积原因，**这些资源未包含在仓库中**。请将它们放置在 `public/` 目录下：

```
public/
├── egg/                    # 第1章：单颗鸡蛋缩放序列
│   ├── frame_0001.jpg
│   ├── frame_0002.jpg
│   └── ...
│   └── frame_0180.jpg      # 共 180 帧
│
├── egg_multiple/           # 第2章：多蛋对焦虚化序列
│   ├── frame_0001.jpg
│   ├── frame_0002.jpg
│   └── ...
│   └── frame_0150.jpg      # 共 150 帧
│
└── crack_egg/              # 第5章：裂壳鸡蛋序列
    ├── frame_0262.jpg      # 起始帧号为 262
    ├── frame_0263.jpg
    └── ...
    └── frame_0561.jpg      # 共 300 帧（262–561）
```

> **命名规范**：文件必须遵循 `frame_XXXX.jpg` 格式（4 位零填充数字）。

### 4. 启动开发服务器

```bash
npm run dev
```

开发服务器将在 `http://localhost:5173` 启动（也可通过 `PORT` 环境变量指定其他端口）。

### 5. 构建生产版本

```bash
npm run build
```

构建产物将输出到 `dist/` 目录。

### 6. 预览生产构建

```bash
npm run preview
```

## 架构亮点

### 滚动架构

页面围绕 **GSAP ScrollTrigger** 构建，混合使用固定与非固定章节：

| 章节 | 滚动行为 | 说明 |
|---------|----------|-------------|
| 第1章（Hero） | **固定（Pinned）** | 单颗鸡蛋 Canvas 擦洗，缩放（0.5× → 10×）+ 渐隐至黑色 |
| 第2章（多蛋） | **固定（Pinned）** | 多蛋 Canvas 擦洗，对焦虚化 + 文字编排 |
| 第3章（白鸡蛋） | 非固定 | 滚动驱动图片宽度（150% → 100%）+ 视差文字 |
| 第4章（光影） | 非固定 | 静态内容章节，渐变标题 |
| 第5章（裂壳） | **固定（Pinned）** | 300 帧裂壳序列 + 规格对比展示 |
| 第6章（幕布） | *挂载于第5章* | 灰色幕布升起 + 产品蛋淡入 + 裁剪路径文字过渡 |

### 帧渲染

- 所有 Canvas 章节使用**预加载图像序列**，根据滚动进度逐帧渲染。
- 图像采用 **cover 适配缩放**，确保在所有视口尺寸下实现全出血视觉效果。
- Canvas 尺寸支持 DPR 感知（上限 2×），在 Retina 屏幕上实现清晰渲染。

### 文字编排

文字元素使用 `position: fixed`，通过 JavaScript 驱动变换：
- **第1章 → 第2章转场**：三行内容线汇聚堆叠，然后一起上升并淡出。
- **第2章**：次要文字分三阶段淡入、上移、淡出。
- **第5章 → 第6章转场**：规格列表淡入并落位；白色文字被升起的幕布裁剪，露出下方的黑色文字。

### 导航栏

- 固定顶部导航栏，带**毛玻璃效果**（`backdrop-filter: blur`）
- **深色/浅色自动切换**：主题跟随当前章节背景色
- **激活链接跟踪**：下划线跟随滚动位置（或点击立即跳转）
- 响应式：在移动端（宽度 `< 768px` 或竖屏比例）隐藏链接

### 加载体验

全屏加载页带进度条，预加载全部 630 帧后才展示页面，确保首次滚动即可流畅擦洗。

## 关键设计决策

### 1. 纯 TypeScript + GSAP，无框架

我们刻意**不使用 React、Vue 等前端框架**。本页面以动画为核心驱动，每一章节都需要对 DOM 操作、固定定位和滚动时机进行精确控制。框架的虚拟 DOM、组件生命周期和响应式模型会带来额外开销，并对 GSAP 的滚动固定和直接元素变换产生不可预知的干扰。最终得到一个运行时小巧、零 UI 框架依赖、且所有动画和布局状态完全透明的架构。

### 2. Vite 作为构建工具

选择 Vite 是因为其**极速的开发服务器**和 **ESM 原生打包**，与我们的轻量、模块优先的架构理念高度一致。Vite 开箱即用地支持 TypeScript 编译、热模块替换（HMR）和资源处理，几乎无需配置，构建管线保持简洁专注。

### 3. 预渲染帧序列替代视频

我们用 **630+ 张预渲染 JPEG 帧** 替代了嵌入视频，通过 Canvas 实现逐帧擦洗。这是深思熟虑的创意与技术决策：
- **逐帧精确控制** —— 每个滚动像素对应一帧，无缓冲、无关键帧寻址、无压缩失真
- **无缝合成** —— Canvas 图层可与 HTML/CSS 覆盖层（文字、模糊、透明度、裁剪路径）实时混合，这是原生视频播放无法实现的
- **可预测的性能** —— 所有帧预加载完毕，消除流媒体固有的网络延迟与卡顿

### 4. Canvas 2D 替代 WebGL

对于本项目的视觉语言（摄影级写实 + 滚动驱动帧擦洗），**Canvas 2D** 是务实的最佳平衡点。WebGL 会引入着色器复杂度和纹理管理开销，而视觉收益有限。2D 的 `drawImage` API 配合 cover-fit 缩放，即可从移动设备到 4K 显示器上高效渲染全出血帧画面。

### 5. GSAP ScrollTrigger 作为统一滚动引擎

ScrollTrigger 被选为所有滚动行为的唯一真相来源，将固定、擦洗和进度回调统一在一个系统中。我们没有将多个轻量库拼接（如 Locomotive Scroll + 独立固定方案），而是采用一个成熟、可靠的引擎，确保从首帧到末帧，每一段文字过渡和每一次布局变化都与滚动位置精确同步。

### 6. 共享固定实现章节间无缝过渡

第5章（裂壳）和第6章（幕布）被融合为**一个连续的固定滚动触发器**。这是核心的叙事设计决策：裂壳序列结束之处，灰色幕布恰好开始升起，两者之间零停顿。观众体验到的是连续的视觉流，而非分段式的"翻页"滚动。

### 7. 预入场渐显作为叙事桥梁

从第4章（明亮、轻盈）到第5章（暗黑、戏剧性）的过渡采用了**预入场渐显**策略——裂壳 Canvas 在固定正式触发之前就开始浮现。这不是简单的间隙修补，而是有意设计的叙事桥梁：当观众仍处于明亮章节时，暗画面已缓慢浮现，创造出平滑的色调过渡，呈现出电影般的质感，而非生硬的切换。

### 8. CSS 自定义属性实现视觉与逻辑解耦

最终产品布局（鸡蛋位置、规格列宽度）完全由 CSS 变量（`--s6-egg-left-x`、`--s6-egg-right-x`、`--s6-spec-width`）控制。这是一个有意的设计系统决策：视觉定位留在 CSS 中，而 TypeScript 中的动画逻辑只读取并应用这些值。设计师可以仅通过修改 CSS 调整布局，无需触碰动画代码。

### 9. TypeScript 保障大规模结构完整性

600 多行核心动画逻辑管理着进度计算、布局数学、滚动触发器和状态转换，跨越 6 个章节，**编译时类型安全**不是奢侈品而是必需品。TypeScript 的严格模式在运行前就能捕获坐标不匹配、进度边界错误和状态转换缺陷。
